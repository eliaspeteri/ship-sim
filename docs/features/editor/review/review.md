# Review Workflow

## Purpose

Defines how submissions are queued, reviewed, and approved.

## Scope

- Submission states
- Reviewer UI
- Approval/reject actions
- Diff visualization
- Reviewer comments

## Out of Scope

- Validation engine
- Compilation details

## Implementation Checklist

- [ ] Submission queue list
- [ ] Diff view between versions
- [ ] Geometry/property diff view
- [ ] Reviewer notes/feedback
- [ ] Approve/reject actions
- [ ] Status surfaced to authors
